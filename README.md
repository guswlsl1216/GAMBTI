# GAMBTI
테스트 기반 게임 취향 분석 및 맞춤 게임 추천 웹 서비스

---

#### 📌 프로젝트 개요
GAMBTI는 사용자의 설문 응답을 기반으로  
**게임 취향을 분석하고, 성향에 맞는 게임을 추천하는 웹 서비스**입니다.

본 프로젝트는 Spring Boot와 React를 활용한 첫 번째 풀스택 팀 프로젝트로,  
**테스트 결과 데이터의 저장·조회·공유 흐름**을 중심으로 구현되었습니다.

---

#### 🛠 사용 기술
- Frontend: React
- Backend: Spring Boot
- Database: MySQL
- External API: Steam API
---

#### 👤 담당 역할

#### 1️⃣ 테스트 결과 처리 API
- 설문 응답 기반 테스트 결과 도출 로직 구현
- 테스트 결과 **저장 / 수정 / 삭제 / 조회 API** 개발
- 사용자별 테스트 결과 관리 구조 설계

#### 2️⃣ 결과 공유 기능
- 테스트 결과 공유 및 조회 API 구현
- 공유 링크를 통한 결과 조회 흐름 설계
- 소셜 공유 기능 연동

#### 3️⃣ 데이터베이스 설계
- 게임 장르 관련 테이블 설계
- 테스트 결과와 추천 게임 데이터 간 관계 정의

#### 4️⃣ 프론트엔드 UI 요소
- 로그인 / 회원가입 UI·UX 구현 (기능 구현 제외)
- 서비스 타이틀 및 파비콘 제작

---

### 🧠 프로젝트를 통해 배운 점
- React와 Spring Boot를 활용한  
  **프론트엔드–백엔드 연동 구조에 대한 이해**
- 테스트 결과 데이터가  
  **저장 → 조회 → 공유**되는 전체 흐름 경험
- API 설계와 DB 구조가 서비스 기능에 미치는 영향 체감

---

### 🔍 아쉬웠던 점 및 개선 방향
- 첫 프로젝트로 기능 구현에 집중하여 사용자 경험과 서비스 확장 관점까지는 충분히 고려하지 못함
- **개선** : 이후 프로젝트에서는 API 설계와 데이터 활용 구조를 더 신중히 고민하게 됨
- 프로젝트 당시에는 기능 구현에 집중하여 트래픽 증가나 데이터 누적에 대한 고려는 충분히 하지 못했습니다.
- **개선** : 이후 프로젝트에서는 서버 구조와 데이터 관리까지 함께 고려하는 방향으로 설계를 확장해보고 싶습니다.
